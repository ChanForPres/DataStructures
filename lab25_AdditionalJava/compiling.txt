The directory structure is the following:
emotions/
|__deep
      |__greetings
 	      |__HelloWorld.java
      |__feelings
	      |__gratitude
		      |__ThankYou.java

Basically folders and packages have no difference. Creating subsequent folders of src-com-utils will be the same as having a package com.utils. 
——————————————————————————————————————————————————————————————————————————————
Example:
|__Exec Jar
      |__classes
      |__src
	   |__com
	       |__g2
                  |__ExecJar
                       |__Person.java
                       |__Start.java

* To compile: 
> javac -d “classes” “src/com/g2/ExecJar/Person.java”
// -d stands for directory. It explains compiler where the class files should be created
// Last argument is class path where the java file exists

> javac -d “classes” -classpath “classes” “src/com/g2/ExecJar/Person.java”

1. What is the sequence of commands you need to do in order to compile and run HelloWorld.java from the folder above emotions/?
* First in the source code, 
	- HelloWorld.java imports gratitude.ThankYou (which signifies that ThankYou.java is in a package called gratitude).
	- Hello	World.java is in a package called greetings

* In order to compile: 
	> javac -cp src emotions/deep/greetings/HelloWorld.java
* In order to run:
	> java -cp emotions.deep .:feelings:greetings HelloWorld
